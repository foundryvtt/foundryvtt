Overview

This article will detail where Foundry VTT stores user data, and some common actions that can be taken with it.

Information contained in this article relates to data that could be lost or damaged if mishandled. Please be careful when manipulating files and folders in detailed below, and be sure to create backups before making any changes.

User Data Locations

Foundry VTT stores User Data separately from the program data in order to keep your games safe from accidental deletion when the program is updating. Depending on operating system, user data is stored in a different default locations.

Default UserData Locations

  • Windows 10 - %localappdata%\FoundryVTT\
  • macOS - ~/Library/Application Support/FoundryVTT
  • Linux - ~/.local/share/FoundryVTT

For guidance on how to format, name and organize your files within the Foundry user Data folder, please refer to the @Article[media] article.

Core Data Folders

When Foundry is first installed it comes with three folders that are core to its data storage: modules, systems, and worlds. Most users will not need to interact with these folders or place files in them under normal circumstances, as they are used by the software to store information.

modules
This folder is where Foundry places modules that are installed through the add-on module menu. Modules will store the majority of their assets here, as well.
systems
This folder is where Foundry places systems that are installed through the game systems menu. Systems will store all of their art, icons, and similar assets here as well.
worlds
This folder is where Foundry places your created game worlds and any external game worlds you install. By default these folders will not contain media assets (art of audio) unless you specifically place them there, or they are included, such as with a premade world that you install.

Duplicating a World

It is possible to duplicate a world in order to run the same content for additional groups or preserve a pristine, unedited version of your world before you start a campaign. To do so, follow these steps:

  1. Close FVTT
  2. In your filesystem, duplicate your world folder inside /worlds in your User Data path..
  3. Rename the new duplicate, without using spaces.
  4. Edit the world.json file inside that folder so that the name entry matches the folder name exactly, this is case-sensitive. Edit the title entry to be the new display name.
  5. Open FVTT.

    6. You should now have a duplicate copy of your world showing in the worlds list.
