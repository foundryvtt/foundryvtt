Overview

This article presents the basics of the Foundry VTT development cycle and what to expect from a Foundry VTT release or Game System based on its version information.

WARNING: You should never attempt to run a live game using anything other than a Stable Release. This is especially true if you use many modules in your games.

  • For information about how to safely test out a new version of Foundry VTT, please see the Updating to a New Major Version of Foundry VTT article.
  • If you are a community developer who is looking for additional information about what to expect during each phase of Foundry VTT development, please see this GitHub Discussion.

Foundry VTT Release Types and Versioning

To understand Foundry releases and versioning, it is important to know the basics of the Foundry VTT development cycle.

At first, new features are sketched in and changes are made freely and rapidly. As we get closer to release, the pace of change slows as the focus shifts more towards refinements and bug fixes. Finally, Foundry VTT is deemed feature-complete and reliable enough to create the first Stable build of the next generation of Foundry VTT.

The Phases of the Foundry Development Cycle

Each time work starts in the development cycle for a new "generation" of Foundry VTT, it is given the next major version number (for example, 13) and is referred to as "Version 13" or "V13" for short.

During this development cycle, each release is categorized by the current phase of the development cycle:

Prototype
The first few releases of the development cycle of a new generation. These "first draft" releases may change wildly and are only really useful for developers who are curious about about the initial thoughts of the Foundry VTT team.
Development
In the development phase, general ideas and approaches solidify a bit more. Focus shifts towards becoming feature-complete and addressing community developer feedback.
Testing
In the Testing phase, the focus shifts to refining the existing major features, bug fixes, and perhaps adding some last minor features.
Curious power users can now start checking things out. Be sure to take a Backup of all your important Worlds first, though!
Stable
Once Stable hits, the new generation is ready for primetime and live games. Make sure to update safely, though!!
A few more "maintenance" Stable releases may be released to refine features and fix any remaining bugs, and then the cycle begins anew.

Understanding Versioning

In the image to the right, Foundry Virtual Tabletop is running Foundry V13 Testing 3, also called 13.338. What does this mean?

Looking at the Foundry VTT version information, we see that this is Foundry VTT V13 Testing 3, meaning that it is the third build in the Testing phase of the 13th generation of Foundry Virtual Tabletop. This user is likely running their live games in Foundry VTT V12 Stable and carefully installed this build to check out the next major Foundry VTT version, V13. They may also do some testing to see how some of their favorite modules are running in it.

The 338 in the screenshot is the exact build number. This release can also be referred to by its generation number and build number (13.338) as a convenient shorthand.

Game System Versioning

Additionally, we see that Foundry VTT is using the D&D 5E game system.

Although versioning methods vary between game systems, generally speaking the first number always indicates the major version of the game system and is incremented when major changes are made. If the either the first or second number increments, you should assume that some modules may no longer be compatible with that version until they update. If only the last number increments, a more minor update was likely deployed and most modules should not be affected.
