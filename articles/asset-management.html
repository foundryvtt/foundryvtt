Clever use of multimedia assets such as images, sound, and videos with Foundry VTT can dramatically increase your game's quality and immersion. This article covers:

  • Locating assets
  • Uploading assets so that you can use them inside Foundry VTT
  • Properly backing up your assets
  • Additional detail for power users

Locating Assets to use in Foundry VTT

If you are using a pre-made adventure, most assets that you need are included in the module and are pre-configured for your use. Available adventures include:

  • Demon Queen Awakens (D&D 5E and PF2E)
  • A House Divided (D&D 5E)
  • Ember (D&D 5E and Crucible)
  • Phandelver and Below (D&D 5E)
  • Tomb of Annihilation (D&D 5E)
  • Beginner Box (PF2E)
  • Abomination Vaults (PF2E)
  • Kingmaker (PF2E)

Many more adventures are available for a wide variety of game systems. For more information, try searching on the official Foundry VTT Marketplace or asking on our Discord (https://discord.gg/FoundryVTT) in your favorite game system's channel.

Finding additional assets

Even when using a pre-made adventure, it can be useful to add your own assets (player token and profile images, for example). Most modern file formats can be used in Foundry VTT, such as .png, .jpg, .webp, .webm, .mp4, .mp3, and .ogg.

Note: Despite some lingering popularity, .gif is an obsolete, inefficient file format and is not supported.

This means that there is a wealth of high-quality art available to support your games, both free and paid. Creators such as Forgotten Adventures have an extensive library of free assets. If you're looking for something in particular, try visiting the #media-and-assets channel of our Discord (https://discord.gg/FoundryVTT), checking the pins, and asking around.

Of course, the Foundry VTT Marketplace is also an excellent resource for high quality TTRPG-ready assets of all kinds.

Creating and optimizing assets

If you are creating your own assets to use or using existing ones, following some media best practices can drastically reduce file sizes. This can save both storage space and upload times. For more information, see the Media Optimization Guide.

"Uploading" Assets

Once you've found a file that you like, you can upload files in one of two ways:

  • Using your operating system
  • Using Foundry VTT

"Uploading" a file with your operating system

You can always use your operating system to move files into the Foundry User Data folder without using Foundry VTT at all. Simply move the files using Windows Explorer, macOS Finder, or standard Linux commands. If you are self-hosted, it seems a bit strange to call this movement of files on the same "uploading," but this makes the files available for use within Foundry VTT.

For more information about the Foundry VTT User Data folder, please see the Configuration article.

Troubleshooting Tip: Why is my upload button disabled?

As of Version 13, users can now only upload files into specific folders. This change was made to prevent accidentally overwriting files in certain cases and to encourage better file management, but it can be a bit confusing if you're not expecting it.

If you want to upload a file and the Upload button is disabled, hovering over it provides some additional instructions:

Choosing a different folder to upload into will resolve this issue for you. If you see an "assets" folder at the topmost level of your User Data folder, you can always upload into that folder. You can also always upload into the folder of the world that is currently running.

If you create a new top-level folder in your user data folder (next to the assets, modules, systems, and worlds folders), you will also always be able to upload into that folder and its sub-folders.

Troubleshooting Tip: Why can't my players upload files?

By default, players cannot upload files.

If you want to give them permission to upload their own files onto your server, you will need to configure your user permissions first so that they have the Upload New File permission. For more information, see the Users and Permissions article.

Assets and Backups

Foundry VTT has a built-in backups feature that can be very helpful in recovering your data or reverting a world to a past state. Occasionally, after restoring a backup, a user is surprised by one or more broken links to their assets.

When you back up a world, only the contents of the World are backed up. This means that any assets that are stored in different locations are not backed up each time a backup is taken. This saves significant amount of hard drive space, but you need to ensure that any assets used in the world are in the same relative position that the backup expects.

For example, if you had a token image saved directly inside the assets folder, but later reorganize your files and move it to a new assets\tokens folder, when you restore to the older backup the token's image will be a broken link.

Storing all assets that you use inside your Foundry worlds in a single folder (like the assets folder) helps you keep all your assets together, making it easier to periodically take a manual backup copy of these images.

Additional Power User Details

Manually allowing or prohibiting uploads in certain folders using canUpload

The mechanism for prohibiting uploads into individual world, module, and system folders is the canUpload line inside the folders's corresponding world.json, system.json, or module.json file:


      "flags": {
    "canUpload": true
  },

If the canUpload key is omitted it is considered false by default.
