Overview

This article is intended to provide guidance on the preferred way to create and revise KB articles. In cases where multiple approaches seem reasonable (such as referring to the software as "Foundry" vs. "Foundry VTT" or "FVTT") the "correct answer" can be looked up here.

General Guidelines

Be concise
 
If reasonable, have a catchy splash image/animation and intro
 
Avoid large walls of text
 
Use a personable tone
 
Use semantic HTML
 
Keep accessibility in mind
 
Use collapsible sections
 
Prefer definition lists wherever possible over ordered and unordered lists
Definition lists are more semantic and have preferable formatting.

Element-specific guidelines

Images

Use figures and captions
When placing images, use <figure> and <caption> tags to do so.
Add alt text

Headings

Borders
Major headings should have the "border" class
Every heading should have an ID tag

Branding

"FVTT" should never be used in official documentation. Instead, "Foundry" or "Foundry Virtual Tabletop" is preferred.

Capitalization

Foundry Documents should always have their name capitalized (e.g., Actor, Scene)
