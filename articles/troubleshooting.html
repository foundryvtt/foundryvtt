Overview

This article contains general tips and best practices to use to troubleshoot unexpected Foundry behaviors. It also contains detailed information about a number of common issues and their solutions.

General Troubleshooting Tips

Initial Troubleshooting

When first encountering an unexpected behavior, there are some tried-and-true techniques that can resolve many issues easily or at least help further narrow down what is happening.

Refresh the affected web browser and/or Foundry
Occasionally a GM's Foundry client can get out of sync with a player or another similar temporary issue can crop up. Have affected users hard refresh Foundry or their browsers by pressing `Ctrl-F5` and seeing if that resolves the issue.
Restart the Foundry server
Sometimes a full Foundry server restart can resolve a temporary issue as well.
Disable all modules
Often, a module or a combination of modules is the root cause of an issue rather than Foundry itself. If the issue no longer occurs with all modules disabled, it is a module-related issue. Try enabling modules slowly again until you determine which module (or combination of modules!) is responsible.

To resolve the module issue, try updating to the latest version of the module. If that does not work, try visiting the module's GitHub or GitLab page or or joining our Discord community and heading to the #module-troubleshooting channel.
Try an "incognito" browser or use a different browser
Sometimes, a web browser's settings or extensions can cause issues. To eliminate this possibility, pick a different supported web browser (Note: Opera GX and Safari are not currently supported!). For example, connect using Firefox instead of a Chromium-based browser (Chrome/Opera/Edge), or pick a Chromium browser to try if you are already using Firefox. Using incognito mode delivers many of the same benefits while being a bit easier.

Launch the world in Safe Configuration
If you are having trouble opening your world at all, try launching the world in Safe Configuration.
Ask for help
If you are running into issues which are not resolved here, we strongly recommend that you contact us or join our Discord community for live support.

Passwords and Account Recovery

Troubleshooting Specific Issues

Reset Your Administrator Password

How to Reset Your Foundry Administrator Password

This article resolves any of the following issues:

  • I can't remember my admin password.
  • I can't use the return to setup feature or log in to Foundry VTT.
  • I am seeing a red warning message about resetting my admin password.
  • I am seeing the following error message:

The provided server administrator password is invalid. If you have forgotten your configured password you will need to manually delete the Config/admin.txt file from your user data location.

How to Reset Foundry User Passwords

Regain Access to your Foundry VTT Website Account

If you are not receiving an email after using the reset password link in the Foundry website, the usual cause is that the email you are checking is not the email that is associated with the account. Try to think of any other email accounts that your Foundry account could be linked to and check those as well.

If necessary, we can restore access to your account by emailing support@foundryvtt.com. Include as many of the following details as you can:

  1. Your username (not the display name, which is different)
  2. The email address you used to use
  3. The date when you purchased Foundry Virtual Tabletop
  4. Your software license key (if you still know it)
  5. The last 4 digits of the credit card used to make the purchase
  6. The cardholder name of the credit card used to make the purchase
