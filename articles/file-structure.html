Overview

This article will detail preferred file naming practices, where Foundry VTT stores user data, and some common actions that can be taken with it.

Information contained in this article is considered a vital first-step for optimizing files, and something that all users should practice. This will help ensure your files are organized and loading without issue. Improperly organized and named files can make session planning unnecessarily difficult and in some rare cases, cause files to fail to display or load correctly.

File Structure and Naming

This section details the recommended method of naming and organizing your files. Proper organization, file naming, and formatting will ensure Foundry works as smoothly as possible, reduce the risk of issues, and potentially improve your own workflow efficiency.

Folder Structure

It is recommended by the Foundry VTT creator Atropos that you organize by the thematic purpose for different assets:

  • /artwork for art included in journals, items, or handouts
  • /audio for sound files
  • /tiles for tile images used in scenes
  • /tokens for actor tokens
  • /maps for map backgrounds used in scenes

File Naming

Since Foundry VTT is a web server, it adheres to file naming conventions expected for a web server, and thus the file structure and filenames of your assets should be standardized accordingly. The most common things are:

1. Don't include spaces in filenames or folders.

For example: /modules/my awesome content/maps/fortress of doom.jpeg should be avoided. Spaces should either be removed, or replaced with dashes (-) and underscores (_) instead. So the file and folder above would become: /modules/my-awesome-content/fortress_of_doom.jpeg

2. Avoid unnecessary metadata in filenames.

It is common for content creators to include metadata in their file names in order to make the images they're distributing to users easier to identify when there are variations.

An example of this would be fortress_of_doom(30x25-3000x2500-gridless).jpeg which is not inherently wrong, but should be shortened to just fortress_of_doom.jpeg since that extra information is not necessary for a premade module.

3. Don't use potentially disruptive or special characters in filenames or folders.

It's best to stick to universally readable characters to avoid potentially breaking file paths or creating errors within Foundry VTT when reading filenames. This includes spaces (as above), characters specific to non-english keyboards, and the following characters:

a-z, A-Z, 0-9, . - _ ~ * ' ( ) ; : @ & = + $ , / ? % # [ ]

Token File Naming Conventions

If providing multiple varieties of the same base token, file naming conventions should allow all tokens to begin with the same file prefix. This should generally be the creature name, major variations, then minor variation.

Example:

red-guard.png, blue-guard.png, green-sword-guard.png, green-guard-spear.png.

Should be: guard-red.png, guard-blue.png, guard-green-sword.png, guard-green-spear.png instead of

This way, Foundry wildcard token paths can easily use guard-*.png to get a guard of any of the major colors available (red, blue or green) or guard-green-*.png to get a green guard with a random weapon (a sword or spear in this case).

User Data Locations

Foundry VTT stores User Data separately from the program data in order to keep your games safe from accidental deletion when the program is updating. Depending on operating system, user data is stored in a different default locations.

Default UserData Locations

  • Windows 10 - %localappdata%\FoundryVTT\
  • macOS - ~/Library/Application Support/FoundryVTT
  • Linux - ~/.local/share/FoundryVTT

Duplicating a World

It is possible to duplicate a world in order to run the same content for additional groups or preserve a pristine, unedited version of your world before you start a campaign. To do so, follow these steps:

  1. Close FVTT
  2. In your filesystem, duplicate your world folder inside /worlds in your User Data path..
  3. Rename the new duplicate, without using spaces.
  4. Edit the world.json file inside that folder so that the name entry matches the folder name exactly. Case-sensitive. Edit the title entry to be the new display name.
  5. Open FVTT.

    6. If it worked, you should now have a duplicate copy of your world showing in the worlds list.
