Overview

The Configure Controls Panel contains all of the controls currently available to users in the Foundry application and allows you to configure the keyboard commands that are used to manipulate the software. These keybindings can be completely changed from the default to suit whatever workflow you are most comfortable. The workflow for opening, editing, and resetting keybinds is explained in this article.

Configure Controls Panel

From the Game Settings tab ( on the sidebar) the keybind configuration menu can be opened by clicking the Configure Controls button. From this panel users can customize the controls for the core software, plus those of any modules or game systems which have added custom keybinds and made them available to edit. There are only three parts to this panel, the action filter which allows sorting and searching, the action categories which allow for sorting what actions are shown, and the action list, which shows all of the available actions and their keybinds.

Action Filter

You can use the Action Filter to find a specific action that matches the terms entered into the dialog box. This is useful for looking up specific actions in the list.

Action Categories

This lists all the possible filter categories available in the Control Configuration panel. The All Actions category is always selected by default. Clicking on one of the categories will limit the visible actions to only the ones that match the chosen filter, allowing users to configure keybinds only related to that aspect of the software. By default there are only three categories available to users: All Actions, Core Keybindings, and Core Mouse Controls. Additional filter categories may appear in this list if module and game system developers choose to add them.

Action List

The Action List shows all of the available actions in the application and their keybinds and customize them as they see fit. Its features are explained below.

Editing Keybinds

Editing existing keybinds is straightforward and simple using our Keybind UI. To edit an existing keybind, find the action you wish to change and then either click the edit button (), or double click the displayed keybind for the action. This will change the existing keybind to a text field with a keyboard () next to it indicating it is waiting to receive a key press.

While this field is active any keys or key combinations pressed will be displayed in the field as the new, pending key bind. To save this key or key combination simply click the save icon () to make the key choice final. If during this edit process you decide you do not want the current keybind and want to keep the previous one, simple click the delete keybind button (), this will discard the current pending keybind and stop the keybind editing process.

Adding New Keybinds

To add a new keybind click the create new keybinding button () next to the action you want to add a new keybind to. This will create a new empty binding which you can select and define a key. This process is identical to that of editing a keybind. Saving the keybind will add it to the possible keybinds that will trigger that function. There is no upper limit to the number of additional keybinds a function can have.

Deleting & Resetting Keybinds

To delete an additional keybind, simply click the Delete Keybind button (), this will delete the keybind from the action list. Using the Delete Keybind button on a core keybind will not delete it from the list but instead delete any custom keybind that has been set for it.

You can also use the Reset Defaults button at the top of the configure controls panel to reset all keybindings to their default settings. If you have configured any custom key binding you will be asked to confirm this reset so that you don't accidentally lose your custom binds.

Key Conflicts

By default the keybind menu detects possible key conflicts and displays a yellow warning sign () beside any actions that have a potential conflict with another keybind. Hovering your cursor over the yellow warning icon will present a tooltip indicating what keybind(s) are in conflict. Not all conflicts need to be resolved, as the bindings used may not affect each based on what aspects of the software they influence. For example, Move Along Measured Ruler and Pause Game conflict in the binding menu but do not conflict during use of the software.
